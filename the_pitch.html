Solution: Our Endpoints

These descriptor classes simply map an http method, uri, some args, a function and some other nifty things and you're done. Want a new endpoint? Make a new descriptor class. Done and done.

Problem: You got your validation in my function.

Input validation is married to the actual service. How many times do you want to check:


    try:
        i = int(val)
    except:
        throw ValidationError()

or alternately:


    assert isinstance(val, int), "This has to be an int"

Solution: Our Args

You simply have to add an Arg to your Endpoint's args and the input will be validated and in the correct type by the time your functional code receives it. Otherwise an appropriate error is sent back. It's OMG magical! The following is the built-in Int validation that Nudge provides.


    def Int(min_=None, max_=None):
        def f(s):
            try:
                i = int(s)
            except (ValueError, TypeError):
                raise ValidationError("must be a number")
            if (min_ and i < min_):
                raise ValidationError("must be >= %d" % min_)
            if (max_ and i > max_):
                raise ValidationError("must be <= %d" % max_)
                
            return i
        return f
